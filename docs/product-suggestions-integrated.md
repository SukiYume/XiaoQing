## 🎯 功能需求清单

### P0 - 核心功能（立即实施，高ROI）

#### 1. 🔬 数据处理任务监控

**痛点**：射电数据处理往往耗时很长，需要随时了解任务进度，不需要一直盯着终端

**功能设计**：

| 命令 | 功能描述 |
|------|---------|
| `/task add <任务名称/脚本路径>` | 添加任务监控（如 CASA、GILDAS 脚本） |
| `/task status` | 查看所有任务状态 |
| `/task list` | 查看任务队列 |
| `/task unwatch <任务>` | 停止监控 |

**自动化能力**：
- 实时推送任务状态（开始/进度/完成/失败）
- 异常自动报警（内存溢出、磁盘满）
- 日志摘要推送

**使用示例**：
```
用户: /task add calibrate_dataset_20260122.py
Bot:  ✅ 已开始监控 "calibrate_dataset_20260122.py"

# 3小时后...
Bot:  🔔 calibrate_dataset_20260122.py 已完成
      - 耗时：3h 12m 45s
      - 退出码：0
      - 输出文件：/data/results/calibrated.npy
```

**技术实现路径**：
1. 监控日志文件变化 + 进程检查
2. 定期检查指定目录的输出文件
3. 任务完成时通过HTTP回调通知Bot

**价值**：可以离开电脑做其他事，任务失败立即知道并及时处理

#### 2. 📄 论文与文献管理助手

**痛点**：写论文时需要频繁查文献、格式化引用、记录想法，追踪引用链

**功能设计**：

| 命令 | 功能描述 |
|------|---------|
| `/cite <arXiv ID/DOI>` | 获取 BibTeX 格式引用 |
| `/ads search <关键词>` | 快速检索论文（返回前5篇：标题、作者、年份、引用数） |
| `/ads author <姓名>` | 查找某位同行的最新文章 |
| `/ads bib <Bibcode>` | 输入 Bibcode，返回 BibTeX |
| `/paper note <ID> <内容>` | 为论文记录要点、方法、数据 |
| `/writing <章节> <想法>` | 写作灵感箱，按章节分类记录 |
| `/summarize <arXiv ID>` | AI生成论文摘要 |
| `/cite-network <ID>` | 查看引用关系（谁引用了这篇/这篇引用了谁），生成引用网络图 |
| `/related <ID>` | 智能推荐相关文献 |
| `/my_topics` | 管理研究兴趣关键词 |
| `/ref_add <BibTeX>` | 添加引用到文献库 |

**定时推送**：
- 基于关注的关键词，每日推送相关新论文（如每天9点）
- 截稿日期提醒：`/deadline add "ApJ投稿" 2026-02-15`

**使用示例**：
```
用户: /ads search "fast radio burst"
Bot:  📚 arXiv论文搜索结果
      1. "New FRB Detection..." - Zhang et al. 2026, Cited: 42
      2. ...

用户: /cite 2401.12345
Bot:  📎 BibTeX引用:
      @article{smith2024frb,
        author = {Smith, J. and ...},
        title = {...},
        ...
      }

用户: /cite-network 2401.12345
Bot:  📊 引用网络分析
      - 被引用次数：42
      - 引用论文数：35
      - 关键引用来源：Zhang et al. 2025, Li et al. 2026
      [发送引用网络图]

用户: /paper note 2401.12345 这篇用了ML方法分析FRB
Bot:  📝 已记录论文笔记

用户: /writing 引言 强调FRB研究的重要性
Bot:  💡 已添加到「引言」灵感箱
```

**技术实现路径**：
1. 申请 ADS API Token（免费）
2. 使用 `aiohttp` 请求 ADS API
3. 废弃旧的 `ads_deprecated` 插件
4. 复用现有 memo 插件架构存储笔记
5. 集成已有 LLM 生成摘要
6. 引用网络可用 Semantic Scholar API

**价值**：
- 自动追踪相关领域新论文
- AI摘要快速筛选
- BibTeX一键获取
- 快速开始新论文
- 格式检查自动化
- 引用管理简化
- 追溯引用链、发现相关工作

#### 3. 📁 FITS文件快速预览与诊断

**痛点**：需要快速判断数据质量，在移动端预览大型数据集

**功能设计**：

发送FITS文件 → Bot自动处理 → 返回分析报告

**返回内容**：
- 基本信息（维度、数据类型、头信息、观测时间、望远镜）
- 快速可视化（亮度分布、频谱图、位置信息）
- 诊断信息（NaN像素、饱和点、坏像素比例及建议）

**使用示例**：
```
用户: [发送 fits_file.fits]
Bot:  📄 FITS文件分析报告
      - 文件大小：1.2GB
      - 维度：512 × 512
      - 数据类型：float32
      - 观测时间：2026-01-22 08:30:00
      - 望远镜：ALMA

      📊 快速预览
      [发送可视化图片]

      ⚠️ 诊断报告
      - NaN像素：0.02%
      - 饱和像素：0.5%
      - 建议：可能需要去饱和处理
```

**技术实现路径**：
1. 使用 `astropy.io.fits` 读取
2. `matplotlib` 生成预览图
3. `numpy` 统计分析

**价值**：快速判断数据质量，避免无效处理，节省下载检查时间

### P2 - 进阶功能（按需优化）

#### 8. 🔧 天文计算工具箱 (Astro Toolkit) / 科研资源速查

**痛点**：观测和处理数据时频繁需要时间、坐标转换或源可视性查询；经常需要查常用公式、单位转换、天文常数

**功能设计**：

| 命令 | 功能描述 |
|------|---------|
| `/astro time` | 返回当前 MJD、LST、JD、UTC 时间 |
| `/astro obj <源名称>` | 检索 Simbad 返回 RA/Dec (J2000) |
| `/astro vis <源名称>` | 判断今晚该源是否可见（需配置天文台坐标） |
| `/convert <数值> <单位转换>` | 天文单位转换（Jy↔K, parsec↔light-year, AU↔pc 等） |
| `/formula <公式名>` | 常用公式速查（色散量 dm、红移公式、黑体辐射等） |
| `/calc redshift <z值> <计算项>` | 红移相关物理量计算（距离、年龄、光度距离） |
| `/calc_distance <星系1> <星系2>` | 计算星系距离 |
| `/luminosity <光度>` | 计算绝对星等 |
| `/convert_ra_dec <坐标>` | 坐标格式转换（度数↔时角格式） |
| `/const <常数名>` | 天文常数速查（光速、太阳质量、哈勃常数等） |

**使用示例**：
```
用户: /astro time
Bot:  🕐 天文时间
      - UTC: 2026-01-22 11:48:00
      - MJD: 60419.4917
      - JD: 2460419.9917
      - LST: 08:23:15

用户: /astro obj Crab Pulsar
Bot:  🌟 Crab Pulsar (M1)
      - RA: 05h 34m 31.94s
      - Dec: +22° 00' 52.2"
      - Constellation: Taurus

用户: /convert 3 Jy 1.4GHz to K
Bot:  📐 流量转亮温度：3 Jy @ 1.4GHz ≈ 1.22K

用户: /formula dm
Bot:  📜 色散量（DM）公式
      DM = ∫₀ᴸ nₑ dl (pc cm⁻³)
      延迟时间: Δt ∝ DM × ν⁻²

用户: /calc redshift z=0.5 distance
Bot:  🌌 红移 z=0.5 对应
      - 光度距离: 2.86 Gpc
      - 角直径距离: 1.27 Gpc
      - 光行时: 5.0 Gyr
```

**技术实现路径**：
1. 添加 `astropy` 到 requirements.txt
2. 利用 `astropy.time`、`astropy.coordinates`、`astropy.units`、`astropy.cosmology`
3. 可扩展现有 WolframAlpha 功能，添加天文专用计算
